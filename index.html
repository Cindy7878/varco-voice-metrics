Insight

이탈 사용자의 세션 시간(7분 06초)은 완료 사용자(14분 52초)의 절반 이하였고, 캐스팅보드에서 보이스를 캐스팅해본 비율도 65.4%로 완료 그룹(94.0%)보다 크게 낮았다.

상당수가 보이스 적용의 진입점 자체를 발견하지 못한 채, 미리듣기와 페이지 탐색만 반복하다 이탈한 것으로 추정된다 — 드래그 앤 드롭 단일 방식의 인터랙션이 발견성(discoverability) 측면의 진입장벽으로 작용했을 가능성.

이 패턴은 이전 분석 기간(5.24–6.17: 참여율 97.4% vs 53.8%)에서도 동일하게 관찰되어 재현성이 확인됐다.

Key Finding

문단 생성 후 보이스 적용으로 넘어가는 관문에서 신규 사용자는 16.8%가 이탈하는 반면, 재방문 사용자는 2.5%(40명 중 1명)만 이탈한다 — 약 6.7배 격차.

방법을 한 번 익힌 사용자는 사실상 전원이 통과하는 관문이라는 점에서, 기능 자체의 문제가 아니라 최초 사용 시 적용 방법을 발견하는 과정이 병목임을 보여준다.

이 패턴은 이후 모든 구간(생성 성공 → 재생, 재생 → 다운로드)에서도 반복되며, 2차 배포의 보이스 적용 인터랙션 개선이 유효한 방향임을 뒷받침한다.

지표 선택 근거 · 사용 패턴 여러 기간 창으로 소요시간을 재검증한 결과, 신규 사용자는 짧은 텍스트로 빠르게 테스트해보고 (막히면) 이탈하고, 재방문 사용자는 세션을 나눠가며 진득하게 오래 사용하는 경향이 확인됐습니다. 즉 신규의 어려움은 소요시간이 아니라 이탈로 나타나기 때문에, 세그먼트 간 소요시간 비교 대신 창 길이에 강건한 이탈률(비율) 지표를 중심으로 구성했습니다.
후속 확인 B 기간(7.15–8.5, 동일 22일 창)에서 같은 비교를 반복해, 신규 사용자의 문단생성 → 보이스적용 이탈률(16.8%)이 낮아졌는지 확인하면 개선 효과의 직접 증거가 됩니다.
